# rednote-distill

> **小红书博主的成功，是可以被蒸馏的。**

## 蒸馏了什么

不是搬运博主内容，是蒸馏五层策略框架：

| 层次 | 蒸馏内容 |
|---|---|
| **怎么定位** | 取名逻辑、Bio 漏斗、人设三角 |
| **怎么写** | 标题钩子、开头节奏、段落律动、CTA 套路 |
| **怎么栏目化** | 内容栏目、发布节奏、漏斗角色 |
| **怎么火** | 身份钩子、情绪引擎、共鸣机制、平台契合度 |
| **护城河在哪** | 什么能学、什么抄不了、怎么替换成自己的 |

## 工作原理

```
博主链接进来
    ↓
三级提取（web_fetch → 浏览器 → 手动粘贴）
    ↓
五层蒸馏框架
    ↓
结构化蒸馏报告
```

### 内容提取策略

小红书是 JS 渲染的，按优先级自动尝试：

| 方式 | 能拿到什么 | 什么时候用 |
|---|---|---|
| `web_fetch` | Bio、粉丝数、名字 | 默认第一步 |
| 浏览器工具 | 可见笔记标题和正文 | web_fetch 拿不到内容时 |
| 手动粘贴 | 任意内容 | 前两个都碰壁时 |

## 与女娲的关系

[女娲.skill](https://github.com/alchaincyf/nuwa-skill) 蒸馏的是**人的认知操作系统**（怎么想、怎么判断）。

rednote-distill 蒸馏的是**博主的内容策略**（怎么定位、怎么写、怎么火）。

同一个「蒸馏」哲学，不同维度。rednote-distill 可以看作女娲在内容创作领域的垂直应用。

## 局限

- **蒸馏不了直觉** — 框架能提取，灵感不能
- **蒸馏不了私域** — 只看公开内容，直播/社群策略无法完整还原
- **快照而非预言** — 蒸馏的是截止当下的策略，不代表未来方向
- **公开 ≠ 真实** — 只能基于公开信息蒸馏
