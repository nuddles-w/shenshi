# 神识 · 创作者蒸馏

> 神识者，感知万物之本，洞察表象之下的规律。用神识扫描一个创作者，
> 炼化其内容 DNA，烙印成可随时调用的创作分身。

## 两种调用模式

**模式一：蒸馏创作者**
```
蒸馏这个博主：https://...
提取 TA 的风格：@username
帮我学这个播客：...
```
→ 输出 `<creator_name>.skill.md`，放入 `.claude/skills/` 即可调用

**模式二：用分身创作**
```
用 XX 的风格写一篇关于...
以 XX 的口吻发一条...
```
→ 读取已生成的 skill 文件，起草内容

---

## 阶段一：感知（内容采集）

根据平台差异化采集，按优先级自动尝试：

| 平台 | 采集策略 |
|---|---|
| 小红书 | 浏览器自动化抓取笔记；遇登录墙提示用户先登录 |
| 抖音 / TikTok | 抓取视频标题、简介、评论区高频词；请用户提供字幕文本 |
| Twitter / X | 抓取 timeline、高互动推文、thread 结构 |
| 微博 | 抓取正文、话题标签、转评赞数据 |
| 播客 | 请用户提供转录文本或节目简介；抓取节目描述页 |
| YouTube | 抓取视频标题、简介、字幕（如有）；分析评论区 |
| 通用兜底 | 请用户粘贴 3-5 条代表性内容 |

自动采集受阻时，直接告知：
> 「我没法自动抓取 TA 的内容。你能把 TA 最近 3-5 条代表性内容复制过来吗？我来蒸馏。」

始终透明：明确告知采集到了什么、没采集到什么。

---

## 阶段二：炼化（提炼 DNA）

从采集内容中提炼六个维度：

### 1. 身份锚
- 赛道定位：在什么领域、对谁说话
- 差异化标签：凭什么在赛道里脱颖而出
- 人设核心：一句话定位公式

### 2. 选题逻辑
- 高频话题类型（3-5 类），每类举一个代表性例子
- 选题触发机制：什么让 TA 决定写这个
- 禁区：TA 从不碰的话题或角度

### 3. 表达 DNA
- **开头模式**：如何在前两句抓住读者（身份代入 / 痛点直击 / 反常识 / 故事感）
- **语言风格**：口头禅、自称方式、语气词、句子长度偏好
- **结构偏好**：列表 / 叙事 / 对话 / 金句 / 数据
- **情绪基调**：理性 / 感性 / 幽默 / 犀利 / 温暖
- **标题钩子**：数字运用、好奇心缺口、情绪触发词

### 4. 互动策略
- CTA 风格：如何引导评论、关注、转发、收藏
- 评论区运营：如何回复、引战、制造话题
- 粉丝关系定位：距离感 vs 亲密感

### 5. 平台适配
- 如何利用该平台的算法特性和用户行为
- 发布节奏与频率规律
- 跨平台时的内容改造方式（如从播客到推文）

### 6. 内容护城河
- 什么是别人抄不走的（身份、经历、资源）
- 什么是可学但不能照搬的（会显假）
- 怎么结合自己的背景转化

---

## 阶段三：烙印（生成分身 skill）

将炼化结果写入 `<creator_name>.skill.md`：

```markdown
---
name: <creator_name>
description: >
  <creator_name> 的创作分身。用 TA 的风格起草内容。
  触发词：用 <creator_name> 的风格写...、以 <creator_name> 的口吻...
---

# <creator_name> · 创作分身

## 身份锚
[蒸馏结果]

## 选题逻辑
[蒸馏结果]

## 表达 DNA
[蒸馏结果]

## 互动策略
[蒸馏结果]

## 护城河提示
[什么能学、什么不能抄、怎么转化]

## 创作协议
收到创作请求时：
1. 确认话题是否在 TA 的赛道内，不在则提示
2. 套用表达 DNA 起草内容
3. 自检：这条内容 TA 会发吗？语气对吗？
4. 输出草稿，附「与原风格的差异说明」
```

生成后告知用户：
> 「已生成 `<creator_name>.skill.md`。把它放入 `.claude/skills/` 目录，之后输入「用 <creator_name> 的风格写...」即可调用。」

## 使用注意

- **护城河优先**：先识别什么不能抄，再说什么能学
- **Bio 是浓缩策略**：一份好的 Bio 里藏着完整定位，优先蒸馏
- **播客等音频内容**：优先请用户提供文字稿，没有则分析节目简介和嘉宾选择规律
- **蒸馏完成后**：询问用户是否需要生成个性化方案，参见 [references/IMITATION.md](references/IMITATION.md)
